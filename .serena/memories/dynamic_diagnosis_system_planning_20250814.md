# 動的診断テキスト生成システム計画記録

## 作成日時
2025-08-14

## 背景・目的
HAQEIの価値観OS分析において、現在の固定テキストから**動的生成による個別診断システム**への進化。二重価値の実現：
1. **読み物価値**: 人間の複雑さが面白い文学的表現
2. **診断価値**: コックピット統合での有料診断材料

## 現状の問題
- 全ユーザーに同じテキスト表示
- スコアの微細差が反映されない
- 診断材料としての価値不足
- コックピット連携での価値観分析が浅い

## 目指す解決策
### スコアパターン分類システム
```
各OS: 高(2.0-3.0) / 中(1.0-1.9) / 低(0.0-0.9)
組み合わせ: 3³ = 27パターンの診断テキスト
```

### 動的生成エンジン
```javascript
スコアパターン → データベース検索 → 個別化テキスト生成
読み物価値 + 診断価値の両立
```
